Portfolio Optimization and Efficient Frontier

Weights, covariance matrices, Sharpe ratio, and random portfolio search

Reference implementation pattern

double portfolio_return(
    const std::vector<double>& weights,
    const std::vector<double>& expected_returns
) {
    double result = 0.0;

    for (size_t i = 0; i < weights.size(); ++i) {
        result += weights[i] * expected_returns[i];
    }

    return result;
}

Learning objectives

  • Represent a portfolio as a vector of weights.
  • Calculate portfolio expected return and volatility.
  • Understand covariance matrices and diversification.
  • Generate random portfolios and identify high-Sharpe candidates.
  • Understand why portfolio optimization is extremely sensitive to assumptions.

Portfolio weights

A portfolio is a set of assets and weights. If weights sum to 1, the portfolio is fully invested without leverage.

For example, weights [0.5, 0.3, 0.2] mean 50% in asset A, 30% in asset B, and 20% in asset C.

In C++, weights are naturally represented as std::vector.

Expected return

Portfolio expected return is the weighted sum of asset expected returns.

The difficult part is not the formula; it is estimating expected returns. Historical averages are noisy and unstable.

For learning projects, historical average return is acceptable. For serious research, expected-return modelling is a major topic.

Covariance and diversification

Portfolio risk depends not only on individual asset volatilities but also on how assets move together.

Covariance measures co-movement. Correlation normalizes covariance into a value between -1 and 1.

Diversification is strongest when assets are not perfectly correlated.

Portfolio variance

For multiple assets, portfolio variance is wᵀΣw, where w is the weight vector and Σ is the covariance matrix.

This looks abstract, but in code it is just nested loops: multiply weight i, covariance i-j, and weight j, then sum.

The covariance matrix is the engine of mean-variance portfolio analytics.

Sharpe ratio

Sharpe ratio is excess return divided by volatility. It is a simple risk-adjusted performance measure.

Be consistent with annualization. Annual expected return should be paired with annualized volatility.

Sharpe is useful, but not complete. It does not fully describe tail risk, skew, drawdowns, or liquidity.

Efficient frontier

The efficient frontier contains portfolios that are not dominated by other portfolios. A dominated portfolio has lower return for the same risk or higher risk for the same return.

A beginner-friendly way to approximate the frontier is to generate thousands of random portfolios, calculate return and volatility, and keep the best ones.

This is less mathematically elegant than quadratic optimization, but it is excellent for learning and visualization.

Project: Efficient Frontier Generator

This is the build task attached to the lesson. Treat it like a portfolio project: make it compile, test it with known inputs, and write down assumptions.

Build steps

  1. Load or hard-code returns for 3–5 assets.
  2. Calculate average annual return for each asset.
  3. Calculate the covariance matrix.
  4. Generate random weights that sum to 1.
  5. Calculate portfolio return, volatility, and Sharpe.
  6. Track the minimum-volatility and maximum-Sharpe portfolios.
  7. Export portfolio points for visualization.

Extensions

  • Add no-short-selling constraint.
  • Add maximum weight constraint per asset.
  • Add risk-free asset and capital allocation line.
  • Add rolling-window optimization.

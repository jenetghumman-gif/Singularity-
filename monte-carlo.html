Monte Carlo Simulation for Option Pricing

Random paths, discounted payoffs, sampling error, and C++ implementation

Core formula / concept

C_0 = e^{-rT}\,\mathbb{E}[\max(S_T - K, 0)]

Visual model: Monte Carlo flow

Set inputs S,K,r,σ,TDraw random ZSimulate S_TCompute payoffDiscount average

The repeated loop is the model. Each path is weak alone; the distribution is the useful object.

Reference implementation pattern

std::mt19937 gen(42);
std::normal_distribution<double> normal(0.0, 1.0);

for (int i = 0; i < paths; ++i) {
    double z = normal(gen);
    double ST = S0 * std::exp((r - 0.5 * sigma * sigma) * T
                + sigma * std::sqrt(T) * z);

    double payoff = std::max(ST - K, 0.0);
    payoff_sum += payoff;
}

double price = std::exp(-r * T) * payoff_sum / paths;

The idea behind Monte Carlo

Monte Carlo simulation replaces an impossible or complicated expectation with an average over many random scenarios.

For option pricing, each simulated scenario produces a possible future asset price. The option payoff is calculated for that scenario. The average payoff is discounted back to today.

The key insight: one path is meaningless, but thousands or millions of paths form an estimated distribution.

Risk-neutral pricing intuition

Under the risk-neutral measure, expected asset growth uses the risk-free rate r rather than a subjective expected return.

This does not mean investors are risk-neutral in real life. It is a pricing technique that makes discounted expected payoff equal to theoretical option value under model assumptions.

For a European call, the payoff at maturity is max(S_T - K, 0). The present value is exp(-rT) times the expected payoff.

Geometric Brownian motion

A common starting process is S_T = S0 exp((r - 0.5σ²)T + σ√T Z), where Z is a standard normal random variable.

The exponential form keeps simulated prices positive. The -0.5σ² term adjusts for lognormal dynamics.

This is the same underlying process used by the Black-Scholes model, which means a Monte Carlo European call price should converge toward the Black-Scholes price as paths increase.

C++ random number generation

Use std::mt19937 as the generator and std::normal_distribution for standard normal shocks.

Set a seed when you want reproducible results. For example, std::mt19937 gen(42).

For production-grade simulation, random-number quality, parallelization, variance reduction, and reproducibility become serious design topics.

Standard error

Monte Carlo output is an estimate, not an exact value. Standard error measures uncertainty in that estimate.

If the payoff standard deviation is s and the number of paths is N, the standard error is approximately s / sqrt(N), then discounted by exp(-rT).

Increasing paths by 100x only reduces standard error by 10x. This square-root convergence is why Monte Carlo can be computationally expensive.

When Monte Carlo is useful

Monte Carlo is especially useful for path-dependent options, multi-asset payoffs, complex risk scenarios, and simulations where closed-form solutions are unavailable.

For a plain European call under Black-Scholes assumptions, the closed-form Black-Scholes formula is faster and exact under the model.

A good quant knows both: use closed-form formulas for speed and validation, use Monte Carlo for flexibility.

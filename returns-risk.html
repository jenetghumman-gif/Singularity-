Returns, Volatility, Drawdown, and Risk Metrics

The measurement layer behind almost every quant workflow

Reference implementation pattern

double variance(const std::vector<double>& returns) {
    double m = mean(returns);
    double sum = 0.0;

    for (double r : returns) {
        sum += (r - m) * (r - m);
    }

    return sum / (returns.size() - 1);
}

double volatility(const std::vector<double>& returns) {
    return std::sqrt(variance(returns));
}

Learning objectives

  • Convert prices into simple and log returns.
  • Compute mean return, sample variance, volatility, annualized volatility, and drawdown.
  • Understand the difference between price risk, return risk, and portfolio risk.
  • Build a reusable risk summary engine.

Why returns are the base unit

Raw prices are not directly comparable across assets. A move from 100 to 105 and a move from 20 to 21 are both 5%, even though the dollar changes differ.

Returns normalize price changes and let you compare different assets, strategies, and portfolios.

Most quant metrics are built on returns: volatility, Sharpe ratio, beta, covariance, VaR, CVaR, drawdown, and strategy performance.

Simple returns vs log returns

Simple returns are intuitive and directly represent percentage gain or loss for one period.

Log returns are mathematically convenient because they add over time. If prices move from 100 to 105 to 110, the total log return equals the sum of the two period log returns.

For small daily moves, simple and log returns are usually close. For large moves, they differ more.

Volatility as dispersion

Volatility measures how much returns vary around their mean. It is not direction; it is variability.

A high-volatility asset can have strong positive returns, strong negative returns, or both. Volatility alone does not tell you whether the investment was good.

Sample volatility divides by n-1, not n, because it estimates population variance from observed data.

Annualization

If your returns are daily, daily volatility can be annualized by multiplying by sqrt(252), assuming 252 trading days.

Mean returns are often annualized by multiplying daily mean by 252, but compounding is more accurate for longer horizons.

Always state frequency. A Sharpe ratio calculated from daily data must annualize both return and volatility consistently.

Drawdown

Drawdown measures decline from a previous peak. It captures path pain better than volatility.

Two strategies can have the same volatility but very different drawdowns. A strategy with long slow losses can feel worse than a strategy with noisy but quickly recovering returns.

Max drawdown is the worst observed peak-to-trough decline in the sample.

Risk reporting mindset

A good risk report does not show one number. It shows return, volatility, drawdown, tail loss, benchmark comparison, and assumptions.

In C++, build this as a RiskSummary struct so your code can return a clean bundle of metrics instead of printing everything from inside calculation functions.

Project: Risk Summary Engine

This is the build task attached to the lesson. Treat it like a portfolio project: make it compile, test it with known inputs, and write down assumptions.

Build steps

  1. Input a vector of prices.
  2. Compute simple returns and log returns.
  3. Compute average return, sample volatility, annualized volatility, best return, worst return.
  4. Compute running peak and max drawdown.
  5. Return all metrics inside a RiskSummary struct.
  6. Print the output as a clean terminal report.

Extensions

  • Add benchmark comparison.
  • Add rolling 20-day volatility.
  • Add rolling drawdown chart data.
  • Export the summary to JSON.

Good C++ quant work is built from small correct functions.

Reference implementation pattern

#include <iostream>
#include <vector>

double simple_return(double previous, double current) {
    return (current - previous) / previous;
}

int main() {
    std::vector<double> prices = {100.0, 102.5, 101.0, 106.2};

    for (size_t i = 1; i < prices.size(); ++i) {
        std::cout << simple_return(prices[i - 1], prices[i]) << "\n";
    }
}

Learning objectives

  • Understand how a C++ program is structured, compiled, and executed.
  • Use variables, functions, loops, vectors, and standard output in a finance context.

Why C++ matters in quant finance

C++ is used when performance, control, and predictable execution matter. In quantitative finance, this can include pricing engines, risk systems, backtesting engines, market-data systems, and latency-sensitive trading infrastructure.

Python is excellent for research and notebooks, but C++ is often used when the same idea must be made faster, safer, and more production-ready.

The beginner trap is trying to memorize syntax. The better approach is to connect every language feature to a quant task: arrays become price series, functions become pricing formulas, loops become simulations, and classes become instruments or models.

Program anatomy

A basic C++ program usually has include statements, optional using declarations, functions, and a main function. The main function is the program entry point.

In a quant setting, you want small functions that each do one clear calculation. For example, one function calculates a simple return, another calculates average return, and another calculates volatility.

This keeps the code testable. If your volatility result is wrong, you can isolate the volatility function instead of debugging an entire application.

Types for finance

Use double for most beginner numerical finance calculations. Prices, rates, returns, volatilities, and option values are continuous quantities.

Use int or size_t for counts, indexes, number of paths, number of time steps, and loop counters.

Be careful with integer division. In C++, 1 / 2 equals 0 because both values are integers. Use 1.0 / 2.0 or cast to double when you need decimal output.

For money in accounting systems, decimals and fixed-point techniques can be important. For quant modelling and simulation, double is the standard starting point.

Vectors as price series

std::vector is one of the most important containers for quant beginners. It stores a dynamic sequence of numbers: prices, returns, simulated paths, weights, losses, or payoffs.

Vectors know their size, can grow with push_back, and store values contiguously in memory, which is helpful for performance.

A price vector with n prices produces n-1 returns because each return compares two adjacent prices.

Worked example: simple returns

Suppose a stock moves from 100 to 105. The simple return is (105 - 100) / 100 = 0.05, or 5%.

If it then moves from 105 to 99, the next return is (99 - 105) / 105 = -0.0571, or -5.71%.

The denominator matters. Returns are relative to the previous price, not the original price unless you are measuring cumulative return.

Common mistakes

Using int for prices or returns, which removes decimals.

Looping from i = 0 and trying to access prices[i - 1], which causes invalid access.

Dividing by prices.size() when calculating returns, even though there are prices.size() - 1 returns.

Ignoring compiler warnings. Warnings are like risk flags: they often point to bugs before they become serious.

Project: Return Calculator CLI

This is the build task attached to the lesson. Treat it like a portfolio project: make it compile, test it with known inputs, and write down assumptions.

Build steps

  1. Create a vector of historical prices.
  2. Write a function simple_return(previous, current).
  3. Write a function simple_returns(prices) that returns a vector of returns.
  4. Print each return as a percentage.
  5. Add average return.
  6. Add input validation: require at least two prices and reject non-positive prices.

Extensions

  • Add log returns.
  • Read prices from a CSV file.
  • Export returns to a new CSV file.
  • Add unit tests for known price examples.

Black-Scholes and Greeks

Closed-form option pricing and sensitivity analysis

Textbook lesson · 70–90 minIntermediateProject-basedQuiz included

Core formula / concept

C = S N(d_1) - K e^{-rT} N(d_2)

Visual model: Greeks map

S changes → DeltaDelta changes → Gammaσ changes → VegaT changes → Thetar changes → Rho

Greeks turn one option price into a risk profile.

Reference implementation pattern

double norm_cdf(double x) {
    return 0.5 * std::erfc(-x / std::sqrt(2.0));
}

double call_price(double S, double K, double r, double sigma, double T) {
    double d1 = (std::log(S / K) + (r + 0.5 * sigma * sigma) * T)
                / (sigma * std::sqrt(T));
    double d2 = d1 - sigma * std::sqrt(T);

    return S * norm_cdf(d1) - K * std::exp(-r * T) * norm_cdf(d2);
}

What Black-Scholes does

Black-Scholes gives a theoretical price for European options under specific assumptions: lognormal asset prices, constant volatility, constant risk-free rate, frictionless markets, and no early exercise.

It is not a universal truth machine. It is a model. Its value comes from speed, interpretability, and the ability to compare market prices against theoretical assumptions.

In C++, Black-Scholes is a perfect first pricing model because it is compact, testable, and connects directly to Monte Carlo.

Inputs

S is the current underlying price. K is the strike. r is the continuously compounded risk-free rate. sigma is volatility. T is time to maturity in years.

The most dangerous beginner mistake is mixing units. If volatility is annualized, T must be in years. If T is in days, convert it using T = days / 365 or a chosen day-count convention.

Rates and volatility should be decimals, not percentages. Use 0.05 for 5% and 0.20 for 20%.

d1 and d2 intuition

d1 and d2 compress moneyness, volatility, time, and rate effects into standardized values.

d1 is related to Delta for a call. d2 appears in the probability-weighted discounted strike component.

You do not need to memorize intuition perfectly at first. Implement carefully, then test heavily.

Greeks

Delta measures sensitivity to the underlying price. Gamma measures how Delta changes. Vega measures sensitivity to volatility. Theta measures time decay. Rho measures sensitivity to rates.

Greeks are not just textbook metrics. They are the language of options risk. A trader may not only ask whether a book is profitable, but whether it is long gamma, short vega, or exposed to time decay.

In code, Greeks can be implemented analytically or through finite differences. Finite differences are slower but are extremely useful for validation.

Finite differences

A finite-difference Delta bumps S up and down by a small h and calculates (price_up - price_down) / (2h).

This technique generalizes. Bump volatility to estimate Vega, time to estimate Theta, and rate to estimate Rho.

Choose bump sizes carefully. Too large gives approximation error; too small can create numerical noise.

Validation

Validate your Black-Scholes function against known examples, online calculators, or a trusted Python implementation.

Then validate Monte Carlo against Black-Scholes. If Monte Carlo with many paths is far away from Black-Scholes under the same assumptions, your simulation probably has an error.

This cross-model validation is a real quant engineering practice.
